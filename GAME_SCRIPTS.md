## 🎮 标准话术模板

### `SCRIPT-CONT-01` 确认继续同一场游戏
**触发：** 间隔≤1 小时，玩家再次触发游戏

**作用：** 告知玩家继续使用上一场的玩家配置，开始新的一局

```
✅ 检测到距离上一局结束未超过 1 小时，继续同一场游戏！

📊 上一场游戏信息：
● 已进行局数：{completed_rounds} 局
● 参与玩家：{player_names}
● 角色配置：平民 {civilian} 名，卧底 {spy} 名，白板 {board} 名

🎲 现在开始第 {next_round} 局，词语已重新生成！
```

**变量说明：**
- `{completed_rounds}` - 已完成局数（如 2）
- `{next_round}` - 即将开始的局数（如 3，= completed_rounds + 1）
- `{player_names}` - 玩家列表（如 @张三 @李四 @王五 @赵六）

---

### `SCRIPT-CONT-02` 询问是否调整配置
**触发：** 紧接着 `SCRIPT-CONT-01` 后

**作用：** 给玩家机会调整玩家列表或角色配置

```
🔧 是否需要调整？

● 调整玩家：回复"换人"或"调整玩家"，然后 @新的玩家列表
● 调整配置：回复"改配置"或"换配置"，然后说新的配置（如"3 平 2 卧"）
● 继续游戏：回复"继续"或"不调整"，直接开始发词

（10 秒内无回复则默认继续）
```

### `SCRIPT-00` 游戏规则说明
**触发：** 用户选择配置方案后、私聊发词前

**作用：** 向所有玩家说明游戏规则和胜利条件，确保所有人理解玩法

```
📖 游戏规则说明

🎭 角色定义：
● 平民：拿到平民词语，需找出卧底
● 卧底：拿到卧底词语，需隐藏身份
● 白板：无词语，靠观察隐藏身份

🏆 胜利条件：
● 平民阵营（含白板）胜利：所有卧底被投出局
● 卧底阵营胜利：存活平民数 ≤ 存活卧底数
  （⚠️ 注意：不是平民全灭！当平民人数≤卧底人数时卧底即获胜）

📋 游戏流程：
1. 收到词语后，线下轮流发言描述（不要说词语本身）
2. 发言结束后，线下投票决定投出谁
3. 在群里 @ 出你们投出的玩家，如：投 @张三
4. 重复以上流程，直到一方获胜

⚠️ 注意事项：
● 不要暴露你的词语
● 不要告诉别人你的身份
● 白板玩家请靠观察隐藏自己

准备好了吗？现在开始私聊发词！
```

### ⚠️ 发词规范（重要规则）

**白板规则：**
- 白板属于**平民阵营**
- 白板胜利条件 = 平民胜利条件 = **所有卧底被投出局**
- 白板玩家私聊内容：`************************************************** 白板 **************************************************`
- 白板没有词语，直接告知"你是白板"

**胜利判定规则：**
- **平民阵营胜利**：存活卧底数 = 0
- **卧底阵营胜利**：存活平民数 ≤ 存活卧底数（⚠️ 不是平民全灭！）
- **游戏继续**：存活平民数 > 存活卧底数 且 存活卧底数 > 0

> - `{civilian_b}` = player_count - 2（2 卧底）
> - `{spy_b}` = 2
> - `{civilian_c}` = player_count - 2（1 卧底 +1 白板）
> - `{spy_c}` = 1

### `SCRIPT-END-02` 游戏结束 - 卧底胜利
**触发：** 存活平民数 ≤ 存活卧底数

⚠️ **注意：不是平民全部出局！** 当存活平民人数小于等于存活卧底人数时，卧底即获胜。

示例：
- 2 平民 +2 卧底 → 卧底胜（2≤2）
- 1 平民 +2 卧底 → 卧底胜（1≤2）
- 3 平民 +1 卧底 → 游戏继续（3>1）

```
🏁 游戏结束！

{player_list}

```

> - `{player_list}` 格式（遍历所有玩家）：

## 📊 场与局的关系说明

### 定义
- **一场游戏（Session）**：一次完整的游戏会话，包含多局游戏
- **一局游戏（Round）**：单次完整的游戏过程（发词→投票→胜负）

### 关系
```
一场游戏 = 多局游戏

19:00 第一场游戏开始
  ├─ 19:00 第一局（卧底胜）
  ├─ 19:06 第二局（平民胜）
  ├─ 19:13 第三局（卧底胜）
  └─ 19:20 第四局（平民胜）
20:25 距离上一局结束超过 1 小时 → 第一场游戏结束
20:30 第二场游戏开始（需要重新@玩家）
```

### 1 小时间隔判断
```
上一局结束时间：T_end
当前触发时间：T_now

间隔 = T_now - T_end

间隔 ≤ 3600000ms (1 小时) → 同一场游戏，继续配置
间隔 > 3600000ms (1 小时) → 新的一场游戏，重新@玩家
```

### 话术分支
| 场景 | 间隔 | 话术 | 操作 |
|------|------|------|------|
| 连续游戏 | ≤1h | `SCRIPT-CONT-01` + `SCRIPT-CONT-02` | 使用上局配置 |
| 新的一场 | >1h | `SCRIPT-03` | 重新@玩家 |
| 首次游戏 | 无记录 | `SCRIPT-03` | 重新@玩家 |
